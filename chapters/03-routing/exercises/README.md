# 第3章 演習問題 - ルーティングとHTTPメソッド

## 問題1: GET メソッド
全ユーザーを取得するGETエンドポイントを作成してください。

## 問題2: POST メソッド
新しいユーザーを作成するPOSTエンドポイントを作成してください。

## 問題3: PUT メソッド
ユーザー情報を完全に更新するPUTエンドポイントを作成してください。

## 問題4: PATCH メソッド
ユーザー情報を部分的に更新するPATCHエンドポイントを作成してください。

## 問題5: DELETE メソッド
ユーザーを削除するDELETEエンドポイントを作成してください。

## 問題6: APIRouter の基本
APIRouterを使って /api/v1/users エンドポイントを実装してください。

## 問題7: 複数のルーター
usersとproductsの2つのルーターを作成し、メインアプリに統合してください。

## 問題8: タグとメタデータ
タグを使ってエンドポイントをグループ化してください。

## 問題9: パスオペレーションの設定
summary, description, response_descriptionを設定したエンドポイントを作成してください。

## 問題10: RESTful API設計
ブログ記事のCRUD操作を実装したRESTful APIを設計してください。
- GET /posts - 全記事取得
- GET /posts/{id} - 記事詳細取得
- POST /posts - 記事作成
- PUT /posts/{id} - 記事更新
- DELETE /posts/{id} - 記事削除
